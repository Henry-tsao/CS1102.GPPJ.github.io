    • How HTTPS Secures the Internet

    HTTPS (Hyper Text Transfer Protocol Secure) is a secured version of HTTP (Hyper Text Transfer Protocol). It uses TLS (Transport Layer Security) or SSL (Secure Socket Layer) to digitally sign the data transmission and encrypt the data transmission between the request side and the response side. HTTPS is more secure than HTTP.

    Features of HTTP & HTTPS

    HTTP

    • Support client/server mode.
    • Simple and fast: When customers request services from the server, they only need to request a method and path. Common request methods include GET, HEAD, and POST.
    • Flexible: HTTP allows data objects of any type to be transmitted.
    • Connectionless: Only one request can be processed per connection.
    • Stateless: HTTP is a stateless protocol.

    HTTPS

    • Content encryption: The content from the browser to server is transmitted in encrypted form, and the intermedary cannot directly view the original content.
    • Authentication: By establishing an SSL(Secure Sockets Layer)/TLS(Transport Layer Security) secure channel between the client browser and the Web server, the transmitted data is encrypted and hidden. Ensure that data is not tampered or stolen during transmission, and ensure data integrity and security.
    • Data integrity: Prevent content from being impersonated or tampered with by third parties.
    • HTTPS is considered a basic requirement that uses SSL/TLS encryption to protect sensitive information like login credentials and credit card details from unauthorized access.
    • HTTPS is indicated by a padlock icon in the web browser's address bar and the "https://" prefix in the website's URL.

    Real-World Applications

    • HTTPS: Bank account login, any website that requires user login.
    • HTTP: Transmitting HTML documents, pictures, videos, etc.
